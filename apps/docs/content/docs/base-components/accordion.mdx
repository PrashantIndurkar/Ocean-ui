## Overview

The Accordion component allows users to toggle the display of sections of content. It's built with Ark UI primitives and styled with Tailwind CSS for a modern, accessible experience.

## Features

- **Accessible**: Built on Ark UI with full keyboard navigation and ARIA support
- **Customizable**: Styled with Tailwind CSS utility classes
- **Type-safe**: Full TypeScript support
- **Flexible**: Supports single or multiple open items

## Usage

```tsx
import {
  Accordion,
  AccordionContent,
  AccordionItem,
  AccordionTrigger,
} from "@ocean-ui/react";

export function Example() {
  return (
    <Accordion defaultValue={["item-1"]}>
      <AccordionItem value="item-1">
        <AccordionTrigger>Product Information</AccordionTrigger>
        <AccordionContent>
          <p>Content for item 1</p>
        </AccordionContent>
      </AccordionItem>
      <AccordionItem value="item-2">
        <AccordionTrigger>Shipping Details</AccordionTrigger>
        <AccordionContent>
          <p>Content for item 2</p>
        </AccordionContent>
      </AccordionItem>
    </Accordion>
  );
}
```

## Props

### Accordion (Root)

The root component that wraps all accordion items.

- `defaultValue` - Array of item values that should be open by default
- `value` - Controlled value (array of open item values)
- `onValueChange` - Callback when value changes
- `collapsible` - Whether items can be collapsed (default: `true`)
- `multiple` - Whether multiple items can be open at once (default: `false`)

### AccordionItem

An individual accordion item.

- `value` - Unique value for this item (required)
- `disabled` - Whether the item is disabled

### AccordionTrigger

The clickable trigger for an accordion item.

- `asChild` - Render as child element
- All standard button props

### AccordionContent

The expandable content area.

- `asChild` - Render as child element
- All standard div props

## Accessibility

The Accordion component follows WAI-ARIA guidelines:

- Keyboard navigation (Enter, Space, Arrow keys)
- ARIA attributes for screen readers
- Focus management
- Proper heading hierarchy
